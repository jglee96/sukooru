# Security Policy

## Supported Versions

Security fixes are applied to the latest published package versions on `main`.

| Version line | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

## Reporting A Vulnerability

- Prefer GitHub's private vulnerability reporting flow from the repository Security tab when it is available.
- If private reporting is not available, open a minimal GitHub issue asking for a private contact path and do not include exploit details, secrets, or proof-of-concept payloads in public.
- Include the affected package name, version, browser/framework/runtime details, reproduction steps, and impact assessment.

## Response Expectations

- We aim to acknowledge new reports within 5 business days.
- Fix timing depends on severity and reproduction quality.
- Public disclosure should wait until a patch or mitigation is available.
